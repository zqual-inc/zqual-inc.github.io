Database

Database System: a software system to manage collection of data (database).

Data Models

Properties: CRUD (All of the models should have such properties)

  2. read

  4. delete

Types of Models

  1. Relational model
  2. Entity-Relationship model
  3. Object-based data model
  4. more than that…

Database Language (aka SQL)

  1. Data Definition Language(DDL): use to define the database schema (such as create table)
  2. Data Manipulation Language(DML): use to manupulate the schema which you create before
    1. Procedural DML: require user to specify data they need and how to get them
    2. Declarative DML: require user to specify data they need
  3. Data control language(DCL)
  4. Data query language(DQL)

  2. Key

Additional Operations

1. Like operators (注意: 沒有等號)
    %: character matches any string 
    _: character matches any character

    Ex:
    where name like 'Kyrie%' --> Kyrie Irving, Kyrie Lin...
    Match the string "100%"
    where name like '100\%' --> \ 可以忽略%的功能

2. Order by operators (Ordering the tuples)
    Ex: 
    order by name_of_attributes (desc) --> 加了desc變成遞減
    也可以同時order by 好幾個attributes 喔!
    
3. Between Clause
    Ex:
    where salary between 200 and 1000
4. Set Operations: 
    union, intersect, except --> 淺顯易懂
    Ex: 
    (select..from...where..)union (select..from...where..)
    // 會自動把重複地項削掉，加一個 all 就沒事了  
5. Null Value:
    null --> unknown value
    EX:
    select name 
    from instructor
    where salary is null --> 判斷是不是空的

Join Expression

Join Operations: turn two related tables into one
Types:

  1. inner join
  2. left outer join
  3. right outer join
  4. full outer join
  5. natural join

Usually conditions

from table1,table2 == from talbe1 natural join talbe2

Beware of unrelated attributes may combine to one table

1.
select name, title 
from student natural join takes, course
where takes.course_id=course.course_id
2.
select name, title 
from (student natural join takes)join course 
using (course_id)  --> using 用法，接在join 後面
3.
select name, title 
from (student natural join takes)join course 
on takes.course_id=course.course_id -->on的用法和where一樣(有join的情況下才能使用)

Above three examples are the same (1=2=3)

view 沒有group by 或 having 功能
